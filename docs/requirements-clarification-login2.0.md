# 账号登录 2.0 - 需求澄清文档

> **生成时间：** 2026-07-14  
> **当前阶段：** clarify（需求澄清）  
> **技能应用：** brainstorming

---

## 一、需求概述

账号登录 2.0 系统包含 6 大核心功能模块：

| 功能编号 | 功能名称 | 核心特性 |
|---------|---------|---------|
| F1 | 账号密码登录 | 多字段支持、加密传输、记住密码、自动登录 |
| F2 | 手机号验证码登录 | 短信频率限制、自动注册 |
| F3 | 第三方 OAuth 登录 | 微信/支付宝/企业微信 + 账号绑定管理 |
| F4 | 安全防护 | 登录锁定、图形验证码、防暴力破解、CSRF、异地提醒 |
| F5 | 会话管理 | JWT 双 Token、静默刷新、全局登出 |
| F6 | 忘记密码 | 身份验证 → 重置 → 通知 |

---

## 二、各功能模块需求澄清

### F1 账号密码登录

#### 已明确项
- ✅ 多字段支持：用户名/手机号/邮箱三种登录方式
- ✅ 密码加密传输：前端 RSA 加密 + 后端 BCrypt 存储
- ✅ 记住密码：客户端安全存储（可选：加密 Cookie 或 LocalStorage）
- ✅ 自动登录：基于 Refresh Token 的免登录机制

#### 待澄清项
| 编号 | 问题 | 推荐方案 | 风险等级 |
|-----|------|---------|---------|
| F1-Q1 | 多字段登录时，账号类型是用户选择还是系统自动识别？ | **系统自动识别**：通过正则匹配判断（手机号/邮箱/用户名） | 低 |
| F1-Q2 | 记住密码的存储时效？ | **30 天**：符合行业惯例，平衡安全与体验 | 低 |
| F1-Q3 | 密码强度规则？ | **至少 8 位，包含大小写字母 + 数字**：基础安全底线 | 低 |

#### 设计建议
- 登录接口统一设计为单端点，内部根据账号格式自动路由验证逻辑
- 前端密码加密公钥通过专门接口获取，定期轮换（建议 24 小时）

---

### F2 手机号验证码登录

#### 已明确项
- ✅ 短信频率限制：防止短信轰炸
- ✅ 自动注册：验证码验证通过后自动创建账号

#### 待澄清项
| 编号 | 问题 | 推荐方案 | 风险等级 |
|-----|------|---------|---------|
| F2-Q1 | 验证码有效时长？ | **5 分钟**：行业标准，平衡安全与用户体验 | 低 |
| F2-Q2 | 短信发送频率限制规则？ | **60 秒/次，每天最多 10 条**：防止滥用 | 中 |
| F2-Q3 | 自动注册时的默认密码策略？ | **不设置密码**：引导用户后续设置，避免弱密码 | 中 |
| F2-Q4 | 手机号格式校验（大陆/国际）？ | **优先支持大陆手机号**：11 位，国际号码后续扩展 | 低 |

#### 设计建议
- 短信频率限制使用 Redis 实现，Key: `sms:limit:{phone}`
- 自动注册时填充默认用户名（如 `user_{phone}`），引导用户首次登录后完善资料

---

### F3 第三方 OAuth 登录

#### 已明确项
- ✅ 支持平台：微信、支付宝、企业微信
- ✅ 账号绑定管理：支持绑定/解绑第三方账号

#### 待澄清项
| 编号 | 问题 | 推荐方案 | 风险等级 |
|-----|------|---------|---------|
| F3-Q1 | 一个账号最多绑定几个同类型第三方账号？ | **每个平台绑定 1 个**：简化逻辑，避免歧义 | 中 |
| F3-Q2 | 绑定的手机号/邮箱与已有账号冲突时如何处理？ | **提示用户登录已有账号后进行绑定**：避免账号合并风险 | 高 |
| F3-Q3 | OAuth 授权信息存储策略？ | **仅存储 openid/unionid，不存储敏感 token**：符合最小权限原则 | 中 |
| F3-Q4 | 企业微信与普通微信的用户体系是否打通？ | **独立体系**：企业微信面向企业用户，普通微信面向 C 端用户 | 中 |

#### 设计建议
- OAuth 回调地址必须配置白名单，防止 CSRF 攻击
- 账号绑定需二次验证（登录态 + 手机号/密码验证）

---

### F4 安全防护

#### 已明确项
- ✅ 登录锁定：多次失败后锁定账号
- ✅ 图形验证码：防止自动化攻击
- ✅ 防暴力破解：基于 IP/账号双重限流
- ✅ CSRF 防护：Token 校验
- ✅ 异地提醒：检测异常登录地点

#### 待澄清项
| 编号 | 问题 | 推荐方案 | 风险等级 |
|-----|------|---------|---------|
| F4-Q1 | 登录锁定的触发条件和解锁方式？ | **连续 5 次失败锁定 30 分钟**，支持短信验证解锁 | 中 |
| F4-Q2 | 图形验证码的触发时机？ | **首次登录不显示，失败 1 次后显示**：平衡体验与安全 | 低 |
| F4-Q3 | 异地登录的判定标准？ | **IP 所在地与历史登录地不符**：基于 IP 库判断 | 中 |
| F4-Q4 | 异地提醒的通知方式？ | **短信 + 邮件双重通知**：重要账号变更需强提醒 | 中 |

#### 设计建议
- 登录失败记录存储在 Redis，Key: `login:fail:{account/IP}`
- CSRF Token 绑定会话，每个请求刷新

---

### F5 会话管理

#### 已明确项
- ✅ JWT 双 Token：Access Token（短效）+ Refresh Token（长效）
- ✅ 静默刷新：前端无感刷新 Token
- ✅ 全局登出：所有设备同时失效

#### 待澄清项
| 编号 | 问题 | 推荐方案 | 风险等级 |
|-----|------|---------|---------|
| F5-Q1 | Access Token 有效期？ | **30 分钟**：安全与体验平衡 | 低 |
| F5-Q2 | Refresh Token 有效期？ | **7 天**：避免频繁重新登录 | 低 |
| F5-Q3 | 静默刷新的触发时机？ | **Token 过期前 5 分钟** 或 **接口返回 401 时** | 低 |
| F5-Q4 | 全局登出的实现方式？ | **Refresh Token 加入黑名单**：Redis 存储失效列表 | 中 |

#### 设计建议
- Access Token 存储用户基本信息（sub、roles），Refresh Token 仅存储 tokenId
- 静默刷新接口返回新 Token 时，旧 Refresh Token 立即失效

---

### F6 忘记密码

#### 已明确项
- ✅ 流程：身份验证 → 重置 → 通知
- ✅ 支持多种验证方式：手机/邮箱

#### 待澄清项
| 编号 | 问题 | 推荐方案 | 风险等级 |
|-----|------|---------|---------|
| F6-Q1 | 重置密码链接的有效期？ | **15 分钟**：防止链接泄露风险 | 中 |
| F6-Q2 | 新密码是否需要与历史密码不同？ | **不与最近 3 次密码相同**：防止循环使用弱密码 | 中 |
| F6-Q3 | 通知方式？ | **短信 + 邮件**：确保用户及时知晓密码变更 | 低 |
| F6-Q4 | 重置成功后是否需要强制重新登录？ | **强制重新登录所有设备**：防止旧会话风险 | 中 |

#### 设计建议
- 重置链接使用一次性 Token，验证后立即销毁
- 密码变更后发送通知，内容不包含新密码明文

## 三、技术架构确认

### 已验证技术栈（基于 pom.xml 和实施计划）
- **后端框架：** Spring Boot 3.2.0
- **安全框架：** Spring Security 6 + JWT (JJWT 0.12.3)
- **数据库：** PostgreSQL（主库）+ Redis（缓存/会话）
- **消息队列：** 可选 RabbitMQ/Kafka（用于异步通知）

### 数据库设计要点
| 实体 | 核心字段 | 索引建议 |
|-----|---------|---------|
| User | id, username, phone, email, password, status | username, phone, email（唯一索引） |
| OAuthBinding | id, user_id, platform, openid, unionid | (user_id, platform), (platform, openid) |
| LoginLog | id, user_id, ip, location, device, status, created_at | user_id, created_at |
| PasswordResetToken | id, user_id, token, expire_at, used | token（唯一索引） |

## 四、风险与依赖

### 高风险项
1. **OAuth 账号合并逻辑复杂**：需明确设计绑定流程和冲突处理
2. **异地登录判定准确性**：依赖 IP 库数据质量
3. **短信服务商稳定性**：需配置备用通道和降级策略

### 外部依赖
| 依赖项 | 用途 | 可用性要求 |
|-------|------|-----------|
| 阿里云/腾讯云短信服务 | 验证码发送 | 99.9% SLA |
| 微信开放平台 API | OAuth 授权 | 稳定可用 |
| 支付宝开放平台 API | OAuth 授权 | 稳定可用 |
| IP 地理位置 API | 异地登录判定 | 可降级 |
