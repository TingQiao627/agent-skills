# 登录功能 2.0 需求规格说明书

> 版本: 1.0 | 日期: 2026-07-14 | 状态: 需求澄清阶段

## 1. 功能概览

登录功能 2.0 是一个综合身份认证系统，支持多种登录方式和安全防护机制。

| 功能编号 | 功能名称 | 优先级 | 复杂度 |
|---------|---------|-------|-------|
| F1 | 账号密码登录 | P0 | 中 |
| F2 | 手机号验证码登录 | P0 | 中 |
| F3 | 第三方 OAuth 登录 | P1 | 高 |
| F4 | 安全防护 | P0 | 高 |
| F5 | 会话管理 | P0 | 高 |
| F6 | 忘记密码 | P1 | 中 |
| F7 | 辅助功能 | P2 | 低 |

## 2. 详细功能需求

### F1: 账号密码登录

#### 2.1.1 功能描述
支持用户通过账号密码方式登录系统。

#### 2.1.2 核心功能点

| 功能点 | 描述 | 验收标准 |
|-------|------|---------|
| 多字段支持 | 支持用户名/手机号/邮箱作为登录账号 | 系统自动识别账号类型 |
| 加密传输 | 密码传输采用非对称加密（RSA/AES） | 传输层不可见明文密码 |
| 记住密码 | 本地安全存储登录凭证（可选） | 7天内免登录，支持开关控制 |
| 自动登录 | 基于记住密码功能自动完成登录 | 下次打开应用自动登录 |

#### 2.1.3 技术约束
- 密码存储：bcrypt/PBKDF2 哈希
- 加密算法：RSA-2048 或 AES-256-GCM
- 记住密码：Secure Storage（Web）或 Keychain（移动端）

---

### F2: 手机号验证码登录

#### 2.2.1 功能描述
支持用户通过手机号 + 短信验证码方式登录。

#### 2.2.2 核心功能点

| 功能点 | 描述 | 验收标准 |
|-------|------|---------|
| 短信验证码 | 发送6位数字验证码，有效期5分钟 | 到达率 > 95%，延迟 < 10秒 |
| 频率限制 | 同一手机号60秒内只能发送1次 | 超出限制返回友好提示 |
| 自动注册 | 未注册手机号自动创建账号 | 首次登录自动完成注册 |
| 验证码重试 | 最多允许5次验证尝试 | 超出需重新发送验证码 |

#### 2.2.3 技术约束
- 短信服务商：阿里云/腾讯云短信服务
- Redis 缓存验证码（TTL 300秒）
- IP + 手机号双重频控

---

### F3: 第三方 OAuth 登录

#### 2.3.1 功能描述
支持微信、支付宝、企业微信等第三方平台 OAuth 登录。

#### 2.3.2 核心功能点

| 功能点 | 描述 | 验收标准 |
|-------|------|---------|
| 微信登录 | 支持微信开放平台扫码/公众号授权 | 移动端静默授权，PC端扫码 |
| 支付宝登录 | 支付宝账号授权登录 | OAuth 2.0 标准流程 |
| 企业微信登录 | 企业微信扫码登录 | 支持企业内部应用 |
| 账号绑定管理 | 第三方账号与本地账号绑定/解绑 | 支持多账号绑定 |

#### 2.3.3 技术约束
- 协议标准：OAuth 2.0 / OIDC
- 安全要求：State 参数防 CSRF
- 绑定上限：每个第三方平台最多绑定 1 个账号

---

### F4: 安全防护

#### 2.4.1 功能描述
多层安全防护机制，防止恶意攻击。

#### 2.4.2 核心功能点

| 功能点 | 描述 | 验收标准 |
|-------|------|---------|
| 登录锁定 | 连续失败5次锁定账号15分钟 | 支持管理员解锁 |
| 图形验证码 | 失败2次后强制显示验证码 | 支持字母数字/滑块验证码 |
| 防暴力破解 | IP 黑名单 + 行为分析 | 异常登录自动拦截 |
| CSRF 防护 | Token 校验机制 | 所有表单提交校验 CSRF Token |
| 异地提醒 | 检测异常登录地点并发送提醒 | 邮件/短信双通道通知 |

#### 2.4.3 技术约束
- 行为分析：基于 IP、设备指纹、地理位置
- Redis 记录失败次数
- 异地判断：距离上次登录 > 500km 或不同城市

---

### F5: 会话管理

#### 2.5.1 功能描述
基于 JWT 的双 Token 会话管理机制。

#### 2.5.2 核心功能点

| 功能点 | 描述 | 验收标准 |
|-------|------|---------|
| JWT 双 Token | Access Token（短期）+ Refresh Token（长期） | Access 2小时，Refresh 7天 |
| 静默刷新 | Token 过期前自动刷新，用户无感知 | 无感知刷新体验 |
| 全局登出 | 单点登出所有设备 | 管理员可强制踢出 |
| Token 黑名单 | 登出后 Token 立即失效 | Redis 黑名单机制 |

#### 2.5.3 技术约束
- JWT 签名算法：RS256 或 HS256
- Refresh Token 存储：数据库 + Redis
- 双设备登录限制（可配置）

---

### F6: 忘记密码

#### 2.6.1 功能描述
用户忘记密码后的身份验证和重置流程。

#### 2.6.2 核心功能点

| 功能点 | 描述 | 验收标准 |
|-------|------|---------|
| 身份验证 | 通过手机号/邮箱验证身份 | 验证码或链接验证 |
| 密码重置 | 设置新密码 | 密码强度校验 |
| 通知机制 | 密码修改成功后通知用户 | 邮件/短信双通道 |
| 链接有效期 | 重置链接24小时有效 | 过期需重新申请 |

#### 2.6.3 技术约束
- 重置链接：一次性 Token
- 密码强度：至少8位，包含字母和数字
- 历史密码检查：不允许使用最近3次密码

---

### F7: 辅助功能

#### 2.7.1 功能描述
提升用户体验的辅助功能。

#### 2.7.2 核心功能点

| 功能点 | 描述 | 验收标准 |
|-------|------|---------|
| 密码明文切换 | 显示/隐藏密码切换 | 图标切换，默认隐藏 |
| Enter 提交 | 输入框回车自动提交 | 提升操作效率 |
| 登录后重定向 | 登录成功跳转原访问页面 | 保存 redirect_url 参数 |
| 注册入口 | 登录页提供注册链接 | 清晰的引导文案 |

## 3. 非功能性需求

### 3.1 性能要求

| 指标 | 目标值 |
|-----|-------|
| 登录响应时间 | < 500ms |
| 短信验证码到达时间 | < 10s |
| 并发用户数 | ≥ 1000 |
| 可用性 | 99.9% |

### 3.2 安全要求

- 所有接口强制 HTTPS
- 敏感数据加密存储
- 审计日志记录所有登录行为
- 符合 OWASP Top 10 安全标准

### 3.3 兼容性要求

- 浏览器：Chrome 90+, Safari 14+, Firefox 88+
- 移动端：iOS 13+, Android 8+
- 支持 PWA 应用

## 4. 数据模型

### 4.1 用户表（users）

```sql
CREATE TABLE users (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  username VARCHAR(50) UNIQUE,
  phone VARCHAR(20) UNIQUE,
  email VARCHAR(100) UNIQUE,
  password_hash VARCHAR(255),
  salt VARCHAR(64),
  status TINYINT DEFAULT 1,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

### 4.2 第三方账号绑定表（oauth_binds）

```sql
CREATE TABLE oauth_binds (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT NOT NULL,
  platform VARCHAR(20) NOT NULL,  -- wechat/alipay/wework
  openid VARCHAR(100) NOT NULL,
  unionid VARCHAR(100),
  bind_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY uk_platform_openid (platform, openid)
);
```

### 4.3 登录日志表（login_logs）

```sql
CREATE TABLE login_logs (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT,
  login_type VARCHAR(20),
  ip VARCHAR(45),
  device VARCHAR(255),
  location VARCHAR(100),
  status TINYINT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 8. 风险与依赖

### 8.1 技术风险

| 风险 | 影响 | 缓解措施 |
|-----|------|---------|
| 短信服务商故障 | F2 无法使用 | 多服务商降级方案 |

- 短信服务商 API
