## 1. 目标与范围

### 1.1 目标
构建一个可复用的 Skill，自动解析测试结果并生成结构化的标准测试报告，支持 Jest/Vitest/pytest/JUnit XML，提供执行和解析双模式。

### 1.2 成功标准
- AC1: 在 Jest/Vitest 项目中执行"生成测试报告"，产出符合4.2结构的Markdown报告，摘要数据与框架原始输出一致
- AC2: 存在失败用例时，报告失败分析章节包含用例名、文件路径、错误信息
- AC3: 提供JUnit XML文件走解析模式，不触发测试执行即可产出报告
- AC4: 结果文件损坏时，Skill返回明确错误说明而非空报告
- AC5: 覆盖率数据存在时正确呈现，不存在时标注"未获取"且其余章节正常

### 1.3 约束条件
- NFR1: 结果解析与报告生成（不含测试执行）应在5秒内完成（1000用例规模）
- NFR2: 结果文件格式异常时降级输出，不得崩溃
- NFR3: 报告中不得泄露环境变量、密钥类内容
- NFR5: 框架解析器采用插件式结构
- FR1.3: 支持执行模式和解析模式
- FR3.1: 默认输出Markdown（.md）
- FR3.2: 默认输出路径 reports/test-report-<YYYYMMDD-HHmmss>.md

## 2. 技术设计

### 2.1 目录结构
```
skills/standardized-test-report/
├── SKILL.md                    # Skill 主文件
├── scripts/
│   ├── test-runner.js         # 测试执行器
│   └── report-generator.js    # 报告生成器
├── parsers/
│   ├── base-parser.js         # 解析器基类
│   ├── jest-parser.js         # Jest JSON 解析器
│   ├── vitest-parser.js       # Vitest JSON 解析器
│   └── junit-parser.js        # JUnit XML 解析器
└── templates/
    └── markdown-template.md   # Markdown 报告模板
```

### 2.2 核心模块设计

#### 2.2.1 框架检测模块
- 检测优先级：用户指定 > package.json scripts > 配置文件推断
- 支持框架：Jest、Vitest、pytest、通用 JUnit XML

#### 2.2.2 解析器插件架构
```javascript
class BaseParser {
  parse(resultContent) { throw new Error('Not implemented'); }
  getSummary() { return { total: 0, passed: 0, failed: 0, skipped: 0, duration: 0 }; }
  getFailures() { return []; }
  getCoverage() { return null; }
}
```

#### 2.2.3 报告结构
1. 报告头：项目名、生成时间、执行命令、框架/版本、执行环境摘要
2. 结果摘要：用例总数、通过/失败/跳过数、通过率、总耗时、整体结论（✅/❌）
3. 失败用例分析：用例名、所属文件、错误信息、堆栈关键行
4. 用例明细：按测试文件分组的用例列表与耗时
5. 覆盖率：语句/分支/函数/行覆盖率总表，低于阈值的文件清单
6. 附录：原始结果文件路径、生成工具版本

## 4. 风险与缓解措施

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| R1: 各框架 reporter 输出差异大 | 解析器复杂度高 | 采用插件式设计，解析器解耦独立 |
| R2: 测试执行耗时不可控 | 长任务可能超时 | 支持后台执行模式，轮询进度 |

## 5. 验证计划

### 5.1 验证环境
- Jest 项目：使用 sample-jest-project
- Vitest 项目：使用 sample-vitest-project
- JUnit XML：准备样本文件 sample-junit.xml

### 5.2 验证用例
1. **执行模式验证**
   - 在 Jest 项目执行"生成测试报告"
   - 验证报告结构和数据准确性

2. **解析模式验证**
   - 提供已有 JUnit XML 文件
   - 验证不触发测试执行即可生成报告

3. **失败用例验证**
   - 构造包含失败用例的测试结果
   - 验证失败分析章节完整性

4. **覆盖率验证**
   - 构造包含覆盖率数据的结果
   - 验证覆盖率章节正确呈现
   - 构造无覆盖率数据的结果，验证标注"未获取"

5. **异常处理验证**
   - 提供损坏的结果文件
   - 验证返回明确错误说明
