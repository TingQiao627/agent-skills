## 3. 架构设计评审

### 3.1 模块划分

```
test-report-generator-skill/
├── scripts/
│   ├── index.ts          # CLI 入口，流程编排
│   ├── detector/         # 框架检测
│   ├── parser/           # 解析器插件
│   │   ├── base.ts       # 抽象基类
│   │   ├── jest.ts       # Jest JSON 解析
│   │   ├── junit.ts      # JUnit XML 解析
│   │   └── vitest.ts     # Vitest JSON 解析
│   └── generator/        # 报告生成
│       └── markdown.ts   # Markdown 格式化
```

**优点**:
- 清晰的关注点分离
- 解析器插件易于扩展
- 生成器与解析器解耦

**建议**:
- ⚠️ 存在两个 Skill 目录（`skills-test-report-generation/` 和 `test-report-generator-skill/`），需合并避免混淆

### 3.2 数据流设计

```
用户输入 → 框架检测 → 测试执行(可选) → 解析器选择 → 数据归一化 → 报告生成 → 文件落盘
```

设计合理，符合单一职责原则。
