## Overview
实现 T3 级账号登录系统，涵盖账号密码、短信验证码、OAuth 第三方登录、安全防护、会话管理和密码找回六大功能模块。

## Functional Requirements

### F1 账号密码登录
**Scope**: 多字段登录支持、加密传输、记住密码、自动登录

#### 默认推断（无需澄清）
- **登录字段支持**: 用户名 / 邮箱 / 手机号（三选一）
- **加密传输**: HTTPS 强制，密码使用 bcrypt 加盐哈希（cost=12）
- **记住密码**: 
  - 本地存储加密 token（AES-256）
  - 有效期：7 天
- **自动登录**: 
  - 基于记住密码 token 自动续期
  - 设备指纹校验（User-Agent + IP 段）

#### 边界条件
- 用户名长度：4-32 字符，字母数字下划线
- 密码强度：最少 8 位，含大小写字母+数字
- 并发登录：同一账号最多 5 个活跃会话

---

### F2 手机号验证码登录
**Scope**: 短信频率限制、自动注册

#### 默认推断（无需澄清）
- **短信发送限制**:
  - 单次冷却期：60 秒
  - 单日上限：5 条/手机号
  - 单 IP 上限：10 条/天
- **验证码规则**:
  - 长度：6 位纯数字
  - 有效期：5 分钟
- **自动注册**:
  - 新手机号自动创建账号
  - 默认用户名：`user_{手机号后4位}_{时间戳}`
  - 默认密码：空（仅限验证码登录）

#### 边界条件
- 手机号格式：中国大陆 11 位（+86）
- 验证码错误上限：3 次/会话

---

### F3 第三方 OAuth 登录
**Scope**: 微信/支付宝/企业微信 + 账号绑定管理

#### 默认推断（无需澄清）
- **支持平台**:
  1. 微信开放平台（移动应用/Web）
  2. 支付宝（移动应用）
  3. 企业微信（企业内部应用）
- **账号绑定策略**:
  - 一对一：一个第三方账号绑定一个系统账号
  - 解绑限制：至少保留一种登录方式
  - 重复绑定：同一第三方账号不可绑定多个系统账号
- **首次登录**:
  - 自动创建账号（昵称=第三方昵称）
  - 引导绑定手机号/邮箱（可选）

#### 边界条件
- OAuth Token 有效期：30 天
- 企业微信：仅限企业邮箱白名单用户

---

### F4 安全防护
**Scope**: 登录锁定、图形验证码、防暴力破解、CSRF、异地提醒

#### 默认推断（无需澄清）
- **登录锁定策略**:
  - 失败 5 次 → 锁定 30 分钟
  - 失败 10 次 → 锁定 24 小时
  - 锁定粒度：账号级（非 IP 级）
- **图形验证码触发**:
  - 连续失败 3 次后强制显示
  - 有效期：5 分钟
  - 长度：4 位字母数字
- **防暴力破解**:
  - IP 限流：100 次/分钟（全局）
  - 账号限流：10 次/分钟/账号
- **CSRF 防护**:
  - 双重提交 Cookie（SameSite=Strict）
  - 登录表单携带 CSRF Token
- **异地提醒**:
  - IP 地理位置变更超过 500km 触发
  - 通知渠道：短信 + 邮件
  - 通知内容：登录时间、IP、设备信息

#### 边界条件
- 白名单 IP：企业内网段（需配置）
- 验证码错误不计入失败次数

---

### F5 会话管理
**Scope**: JWT 双 Token、静默刷新、全局登出

#### 默认推断（无需澄清）
- **JWT 双 Token 机制**:
  - Access Token:
    - 有效期：15 分钟
    - 存储：内存/SessionStorage
    - 用途：API 鉴权
  - Refresh Token:
    - 有效期：7 天
    - 存储：HttpOnly Cookie（Secure）
    - 用途：刷新 Access Token
- **静默刷新策略**:
  - 前端检测 Access Token 过期时间
  - 提前 5 分钟自动刷新
  - 刷新失败 → 引导重新登录
- **全局登出**:
  - 撤销所有 Refresh Token
  - 清除客户端 Cookie
  - 记录登出日志

#### 边界条件
- Refresh Token 单次使用（用后即焚）
- 并发刷新：仅允许一个刷新请求，其他返回 401

---

### F6 忘记密码
**Scope**: 身份验证 → 重置 → 通知

#### 默认推断（无需澄清）
- **身份验证流程**:
  1. 输入手机号/邮箱
  2. 发送验证码（短信/邮件）
  3. 校验验证码
- **密码重置**:
  - 新密码强度校验（同 F1）
  - 不可与最近 3 次密码相同
  - 重置链接有效期：30 分钟
- **通知机制**:
  - 重置成功：短信 + 邮件通知
  - 通知内容：时间、IP、设备信息
- **安全约束**:
  - 24 小时内最多重置 3 次
  - 重置后强制登出所有设备

#### 边界条件
- 验证码错误上限：3 次
- 企业账号：需管理员审批

## Non-Functional Requirements

### 安全性
- 密码存储：bcrypt (cost=12)
- 敏感数据传输：TLS 1.2+
- Token 签名：RS256（非对称）
- 日志脱敏：密码/验证码不落日志

### 性能
- 登录接口响应时间：< 500ms（P95）
- JWT 验证：< 10ms
- 短信发送延迟：< 3s

### 可用性
- 服务可用性：99.9%
- 降级策略：短信服务不可用时降级为邮件验证码

## Implementation Notes

### 技术栈推荐（基于行业惯例）
- 后端：Spring Boot / NestJS / Django
- 认证框架：Spring Security / Passport.js / django-allauth
- JWT 库：jjwt / jsonwebtoken / PyJWT
- 缓存：Redis（会话/验证码/限流）
- 短信网关：阿里云/腾讯云短信服务
- OAuth SDK：各平台官方 SDK

### 数据库表设计建议
- `users`: 用户基础信息
- `oauth_bindings`: 第三方账号绑定关系
- `sessions`: 会话记录（可选，用于审计）
- `login_logs`: 登录日志
- `password_reset_tokens`: 密码重置令牌
