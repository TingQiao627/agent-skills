# Proposal: 测试报告生成 Skill

## Summary

提供一个 Skill（`test-report-generator`），Agent 在执行测试后能够自动解析测试结果并生成结构化、可读性强的标准测试报告。支持"执行模式"（触发测试运行并收集结果）与"解析模式"（仅解析已有结果文件），默认输出 Markdown 格式报告。

## Motivation

当前团队测试结果散落在终端输出、CI 日志或框架原生产物中，存在以下痛点：
- 测试结果需人工收集、整理、汇总，耗时且易遗漏；
- 缺乏统一格式的测试报告，跨项目/跨团队沟通成本高；
- 失败用例的上下文（错误信息、堆栈、关联代码）需人工回溯；
- 覆盖率、通过率等质量指标无法沉淀为可追踪的历史数据。

## Scope

- 自动识别项目测试框架与运行命令（Jest / Vitest / pytest / JUnit XML 兜底）
- 支持"执行模式"（运行测试 → 收集结果 → 生成报告）与"解析模式"（跳过执行，直接解析已有结果文件）
- 生成标准 Markdown 报告，含：报告头、结果摘要、失败用例分析、用例明细、覆盖率、附录
- 解析器采用插件式架构，支持后续扩展新框架
- 报告默认落盘路径 `reports/test-report-<YYYYMMDD-HHmmss>.md`
- 覆盖率数据可获取时自动呈现，不可获取时标注"未获取"

## Out of Scope

- 不做测试用例的自动生成或修复（仅报告）
- 不做报告的在线托管 / Web 服务化展示
- 不做多次运行结果的趋势对比分析（列为后续迭代候选）
- 不做非测试类质量报告（如 lint、安全扫描）的聚合
- 不自动推送报告到 IM / 邮件等渠道
